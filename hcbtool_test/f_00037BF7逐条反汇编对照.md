## 2. 整体功能概览

从 [`f_00037BF7.lua`](fvp_analysis/result/hcbtool_test/f_00037BF7.lua:1) 看，这一段是“退出游戏时的专用演出函数”，其核心动作不是普通 `Quit`，而是先建立一套全屏 byebye 画面与遮罩层，再按顺序执行淡入/缩放/溶解/退出线程。

已识别到的关键步骤包括：

1. 置 `G[298] = true`，并通过 [`f_00037E46()`](fvp_analysis/result/hcbtool_test/Sakura_hcb_ir/Sakura.lua:120313) 调 `ControlMask(nil)`，屏蔽/冻结用户控制输入。
2. 条件性调用 [`f_00037AE8()`](fvp_analysis/result/hcbtool_test/Sakura_hcb_ir/Sakura.lua:119932)，说明在某些标题/菜单状态下会先做额外清理。
3. 调 [`f_00037B23()`](fvp_analysis/result/hcbtool_test/Sakura_hcb_ir/Sakura.lua:119975)、[`f_00055078()`](fvp_analysis/result/hcbtool_test/Sakura_hcb_ir/Sakura.lua:171526)、[`f_000553AA()`](fvp_analysis/result/hcbtool_test/Sakura_hcb_ir/Sakura.lua:171994)、[`f_0005522A()`](fvp_analysis/result/hcbtool_test/Sakura_hcb_ir/Sakura.lua:171774)、[`f_00054E2E()`](fvp_analysis/result/hcbtool_test/Sakura_hcb_ir/Sakura.lua:171212) 对场景、计时与 dissolve 相关状态进行初始化。
4. 建立 4 个全屏/近全屏 prim：
   - `431`：Tile，全屏遮罩
   - `430`：`menu_byebye_bg`
   - `99`：`menu_byebye`
   - `429`：`menu_byebye` 的另一层，且 blend=1
   - `35`：另一层全屏 Tile
5. 这些图层都被加入 group `797`，随后调用 [`f_000520EE()`](fvp_analysis/result/hcbtool_test/Sakura_hcb_ir/Sakura.lua:165184) 与 [`f_000524BB()`](fvp_analysis/result/hcbtool_test/Sakura_hcb_ir/Sakura.lua:165732) 启动 alpha / scale 运动。
6. 关键动画参数：
   - `430`：alpha `0 -> 160`，时长 `l0`（通常 3000ms）
   - `99` 与 `429`：scale 约 `1100 -> 1000`，时长 `l0`，type=3（减速）
   - `99`：alpha `0 -> 128`，时长 `l0`
   - `429`：alpha `0 -> 128`，时长 `l0`，且 type=1
   - `35`：alpha `0 -> 255`，时长 `l0-500`
7. 最后调用 `ThreadNext` 后以 [`ThreadExit(0)`](fvp_analysis/result/hcbtool_test/f_00037BF7.lua:228) 结束当前线程。

因此，这段函数可以阶段性命名为：

> **ExitGame_Presentation / 退出游戏演出函数**

其视觉效果大致是：

- 先锁定输入
- 叠加 byebye 背景、标志与遮罩
- 通过 alpha 与轻微 scale 收束形成退出演出
- 再通过 Tile 遮罩与 dissolve / fade 把整屏带出
