# LOGO演出解析

## 1. 对象与依据

本文针对 `f_00074DA5()` 这一段 LOGO 演出函数，整理其对几个 `logo_*` 图层执行的具体运动。

主要依据：

- `f_00074DA5.lua`
- `f_00074DA5` 原始 HCB 字节
- `f_00074DA5逐条反汇编对照.md`
- `rfvp` 中对 `MotionAlpha`、`MotionMove`、`MotionMoveR`、`MotionMoveS2`、`MotionMoveZ`、`PrimSetOP` 的参数说明

---

## 2. 图层对应关系

该函数中出现的主要图层如下：

- prim `250` -> `logo_bg`
- prim `252` -> `logo_favo`
- prim `254` -> `logo_favo_view`
- prim `255` -> `logo_favo_view_p1`
- prim `256` -> `logo_favo_view_p2`
- prim `257` -> `logo_favo_view_p3`

可见这是一套由背景、主体 LOGO、前景 view 层，以及三个分片构成的组合演出。

---

## 3. 初始摆位

在动画开始前，脚本先装载并设置这些图层的初始状态：

### 3.1 `logo_bg` / prim 250
- 仅装入资源
- 启动阶段没有明显位移、旋转或缩放设置
- 更像静止背景底板

### 3.2 `logo_favo` / prim 252
- `PrimSetZ(252, 1000)`
- `PrimSetOP(252, 640, 360)`
- `PrimSetAlpha(252, 0)`

含义：
- 初始位于屏幕中心 `(640, 360)`
- 初始完全透明
- Z 层级为 `1000`

### 3.3 `logo_favo_view` / prim 254
- `PrimSetAlpha(254, 0)`
- `PrimSetZ(254, 1000)`
- `PrimSetOP(254, 640, 360)`

含义：
- 初始也位于中心 `(640, 360)`
- 初始完全透明
- 与主体 LOGO 在同一中心位置附近叠放

### 3.4 `logo_favo_view_p1/p2/p3` / prim 255, 256, 257
- 三者都被设为：`PrimSetAlpha(..., 0)`
- 三者都被设为：`PrimSetOP(..., 634, 413)`

含义：
- 三个分片起始锚点相同
- 都从完全透明开始
- 它们后续的视觉差异主要来自动画参数，而不是初始坐标差异

---

## 4. 启动阶段的具体运动

统一时长变量：

- `l0 = 2000`

即这一轮主要动画大都以 `2000ms` 为基准。

### 4.1 prim 257 / `logo_favo_view_p3`
执行了三类动画：

1. `MotionAlpha(257, 0, 255, 2000, nil, nil)`
   - 从透明度 `0` 淡入到 `255`
   - 时长 `2000ms`

2. `MotionMoveR(257, -3600, 0, 2000, 3, nil)`
   - 从 `-3600` 到 `0`
   - 按 FVP 常规理解，旋转单位是 `0.1°`
   - 即：从 `-360.0°` 旋转回 `0°`

3. `MotionMoveS2(257, 5000, 1000, 5000, 1000, 2000, 3, nil)`
   - 宽高缩放参数起止相同
   - 说明它不是“从 A 缩放到 B”的明显变化
   - 更像是在演出期间保持一个固定的非等比缩放状态

### 4.2 prim 256 / `logo_favo_view_p2`
执行了：

1. `MotionAlpha(256, 0, 255, 2000, nil, nil)`
   - `0 -> 255` 淡入

2. `MotionMoveR(256, 3600, 0, 2000, 3, nil)`
   - 从 `+360.0°` 转回 `0°`

3. `MotionMoveS2(256, 5000, 1000, 5000, 1000, 2000, 3, nil)`
   - 与 prim 257 一样，维持固定非等比缩放形态

### 4.3 prim 255 / `logo_favo_view_p1`
执行了：

1. `MotionAlpha(255, 0, 255, 2000, nil, nil)`
2. `MotionMoveR(255, 3600, 0, 2000, 3, nil)`
3. `MotionMoveS2(255, 5000, 1000, 5000, 1000, 2000, 3, nil)`

含义与 prim 256 基本一致：
- 淡入
- 从正向整周旋转收束到正位
- 持续保持一个固定的横向拉伸比例

### 4.4 prim 252 / `logo_favo`
执行了：

1. `MotionMoveZ(252, 750, 1000, 1800, 3, nil)`
   - 这里时长是 `l0 - 200 = 1800ms`
   - 从 Z=`750` 变化到 Z=`1000`

2. `MotionAlpha(252, 0, 255, 1800, nil, nil)`
   - 与 Z 变化同步完成淡入

含义：
- 主体 LOGO 没有明显 XY 平移，也没有旋转
- 它是在中心位置做“纵深推进/靠近”效果，同时完成淡入

### 4.5 prim 254 / `logo_favo_view`
执行了：

1. `MotionMoveZ(254, 750, 1000, 2000, 3, nil)`
   - 与主体类似，也有 Z 轴推进效果

2. `MotionMove(254, nil, 50, nil, 0, 2000, 3, nil)`
   - 参数顺序可理解为：`src_x, src_y, dst_x, dst_y`
   - `src_x = nil`, `dst_x = nil`：X 基本保持当前值
   - `src_y = 50`, `dst_y = 0`：Y 从偏移 `50` 运动到 `0`

3. `MotionAlpha(254, 0, 255, 2000, nil, nil)`
   - 同步淡入

含义：
- 这一层并不是静止出现
- 它是从 Y 方向偏移位置滑回最终位置，同时带一点 Z 轴推进与透明度淡入

### 4.6 prim 250 / `logo_bg`
在启动主动画阶段没有看到它参与旋转、平移或 Z 轴推进。

因此其角色更像：
- 固定背景层
- 等待后续统一淡出

---

## 5. 各图层的视觉分工

把这些运动组合起来，可以得到较清晰的视觉结构：

### 5.1 背景层
- `logo_bg` 负责铺底
- 基本静止

### 5.2 主体层
- `logo_favo` 是主体 LOGO
- 在中心位置淡入
- 同时做轻微纵深推进（Z 轴变化）

### 5.3 前景覆盖层
- `logo_favo_view` 既淡入，又从 Y 偏移位置回到最终位置
- 看起来像一个“观察窗 / 遮罩 / 前景框体”滑入

### 5.4 三个分片层
- `logo_favo_view_p1/p2/p3` 负责制造最显著的动态效果
- 全部淡入
- 255、256 从 `+360°` 回到 `0°`
- 257 从 `-360°` 回到 `0°`
- 三个分片从相反方向旋转收束，形成碎片拼合或徽标聚拢感

---

## 6. 等待与跳过逻辑对运动的影响

动画播出后，脚本会持续检查：

- `MotionAlphaTest(252)`

也就是说：
- prim `252` 的 alpha 动画被用作一段关键演出的完成判据
- 用户可通过输入提前触发 `ControlPulse`，强制进入收尾

若未被跳过：
- prim `250` 会执行一次较长时间的淡出（`3500ms`）
- 说明背景层可能在主标志完成后先慢慢退场

---

## 7. 收尾阶段运动

在收尾阶段，脚本主要做的是统一淡出，而不是复杂运动：

- prim `253`
- prim `252`
- prim `254`
- prim `255`
- prim `256`
- prim `257`

都会执行：
- `MotionAlpha(..., nil, 0, 2500, nil, nil)`

最后还会再做一轮：
- `MotionAlpha(..., nil, 0, 100, nil, nil)`

即：
- 先统一较慢淡出
- 再快速清零透明度

所以收尾的主语言是：
- 整体消隐
- 而不是分层飞散或继续旋转

---

## 8. 运动总结表

| 图层 | prim | 初始位置 | 启动阶段运动 | 收尾阶段 |
|---|---:|---|---|---|
| `logo_bg` | 250 | 未见特别移动，作为背景底图 | 启动阶段基本静止 | 后续淡出 |
| `logo_favo` | 252 | `(640, 360)` | `Alpha 0->255`，`Z 750->1000` | 淡出 |
| `logo_favo_view` | 254 | `(640, 360)` | `Alpha 0->255`，`Y 50->0`，`Z 750->1000` | 淡出 |
| `logo_favo_view_p1` | 255 | `(634, 413)` | `Alpha 0->255`，`R 3600->0` | 淡出 |
| `logo_favo_view_p2` | 256 | `(634, 413)` | `Alpha 0->255`，`R 3600->0` | 淡出 |
| `logo_favo_view_p3` | 257 | `(634, 413)` | `Alpha 0->255`，`R -3600->0` | 淡出 |

补充：
- 255/256/257 三者都还带有一组 `MotionMoveS2(5000,1000 -> 5000,1000)` 参数
- 从脚本字面值看，起止缩放相同，因此更适合解释为“保持固定非等比缩放状态”，而不是明显缩放过渡

---

## 9. 最终结论

这个 LOGO 演出函数对各图层做的核心动作可以概括为：

1. **背景层静止铺底**
2. **主体 LOGO 在中心淡入，并伴随轻微纵深推进**
3. **前景 view 层从下方向上滑入，同时淡入并伴随轻微纵深推进**
4. **三个分片从相反方向整周旋转收束到正位，并同时淡入**
5. **最后全体统一淡出**

因此，它的整体视觉语言不是单纯的“淡入 LOGO”，而是：

> **背景稳住，主体推进，前景滑入，三分片旋转聚合，最后整体消隐。**

这是一个相当典型的启动 Logo / 品牌标识演出脚本。
