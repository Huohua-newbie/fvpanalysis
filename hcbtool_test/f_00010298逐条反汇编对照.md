## 8. 对 `f_00010298` 的整体伪代码总结

把上述逻辑尽量保守地重写，可得到：

```lua
function f_00010298(a0, a1, a2)
    f_00096D1B()

    if G[15] ~= true then
        G[15] = nil
        return
    end

    if a1 == nil then a1 = 1 end
    if a2 == nil then a2 = 0 end
    a0 = nil

    -- 根据 G[1950] 与 G[164] 推导 a0
    if G[1950] == 0 then
        if G[164] == 1 or G[164] == 2 then a0 = nil end
    elseif G[1950] == 1 then
        if G[164] == 1 then a0 = true
        elseif G[164] == 2 then a0 = nil end
    elseif G[1950] == 2 then
        if G[164] == 1 then a0 = nil
        elseif G[164] == 2 then a0 = true end
    elseif G[1950] == 3 then
        if G[164] == 1 or G[164] == 2 then a0 = true end
    end

    if G[1227] == true then
        G[1227] = nil
        f_00055946(1300, nil, nil)
    end

    if a0 ~= nil then
        if G[164] == 1 then
            if a3 == nil then a3 = 550 end
            f_00055D3D(a2, a3, nil, nil, nil, nil, nil, nil, nil)
        else
            if a3 == nil then a3 = 675 end
            f_00055D3D(a2, a3, nil, nil, nil, nil, nil, nil, nil)
        end
    elseif a1 == 1 then
        if G[14] == 1 then
            if a3 == nil then a3 = 550 end
            f_00055D3D(a2, a3, nil, nil, nil, nil, nil, nil, nil)
            G[14] = 0
        elseif G[164] == 1 then
            if a3 == nil then a3 = 550 end
            f_00055D3D(a2, a3, nil, nil, nil, nil, nil, nil, nil)
        else
            if a3 == nil then a3 = 675 end
            f_00055D3D(a2, a3, nil, nil, nil, nil, nil, nil, nil)
        end
    else
        f_00055D3D(a2, 1, nil, nil, nil, nil, nil, nil, nil)
    end

    G[15] = nil
    return
end
```

需要注意：

- Lua-like 导出中的 `a3` 超出函数头参数数目，这和前面 CG 包装函数一样，仍然是导出器的参数槽命名偏移现象。
- 因此这里更应该理解为：函数内部除了 3 个显式入口参数外，还读取了相邻栈槽中的调用上下文值。

## 9. 在 CG 引入流程中的位置

前面的 CG 包装函数几乎都长这样：

```lua
G[flag] = true
f_00010271()
f_00051420(resource_name, ...)
f_00010298(nil, nil, p6)
return
```

据此可以很明确地给 [`f_00010298()`](fvp_analysis/result/hcbtool_test/Sakura_hcb_ir/Sakura.lua:29117) 下定义：

- 它不是“显示某张图”的入口
- 它也不是“清场初始化”
- 它是**显示结束后，根据当前模式与若干全局状态，决定如何触发后续 dissolve / alpha / 等待 / 收尾动画** 的公共后处理层

更准确一点可以表述为：

> [`f_00010298()`](fvp_analysis/result/hcbtool_test/Sakura_hcb_ir/Sakura.lua:29117) 是 CG 显示包装函数族的统一后处理与过渡控制函数，而真正的动画/过渡执行核心则更多下沉在 [`f_00055D3D()`](fvp_analysis/result/hcbtool_test/Sakura_hcb_ir/Sakura.lua:173250) 之中。
