# Sakura 第三个 function（`f_00000872`）说话人名显示详解

## 0. 解析对象

本文解析用户给出的第三个“说话人名显示函数”：

```lua
function f_00000872(a0, a1, a2)
```

从 Lua 片段与原始字节串看，这个函数与前两篇文档中的函数属于同一类：

- 先设置当前说话人/名字栏样式相关全局变量
- 再调用 `f_000019E0` 初始化名字栏显示样式
- 再根据一个模式参数决定显示哪个名字
- 通过 `f_0004CCDA` 真正把名字写入左侧名字栏
- 再调用 `f_000977B8` 做角色状态同步
- 最后把一组上下文变量写入全局区，缓存当前说话人属性

不过这一段比前两段更复杂，因为它不是二选一或三选一，而是一个 **四分支名字显示函数**：

1. `？？？ / ？ソル`
2. `一磨 / ？一磨`
3. `遠矢 / ？遠矢`
4. `ソル`

因此它更像是：

> **“索尔 / 一磨 / 遠矢 这一组身份别名系统的名字栏控制函数”**

为了便于后续统一建模，本文暂命名为：

```text
ShowSpeakerName_SolAliasGroup(...)
```

## 2. 字符串解码与四种显示结果

根据用户给出的 Lua 对照，结合字节串中的 `push_string`，本函数涉及四组关键字符串：

### 2.1 未公开显示
```text
"　 ？？？ 　"
"　 ？ソル 　"
```

含义：
- 左侧名字栏显示未知名 `？？？`
- 内部备用/真实名保存为 `？ソル`

---

### 2.2 第一种别名显示
```text
"　　一磨　　"
"　 ？一磨 　"
```

含义：
- 正常显示名为 `一磨`
- 备用/带问号名为 `？一磨`

这说明脚本并不总是把“一磨”当作完全公开的实名，而是保留了“半公开”表示能力。

---

### 2.3 第二种别名显示
```text
"　　遠矢　　"
"　 ？遠矢 　"
```

含义：
- 正常显示名为 `遠矢`
- 备用/带问号名为 `？遠矢`

---

### 2.4 默认真实名显示
```text
"　　ソル　　"
```

含义：
- 正常显示名为 `ソル`
- 第二参数为 `nil`，说明此时没有额外的备用名需要记录

## 3.1 阶段一：设置当前名字样式编号并初始化名字栏样式

Lua 片段：

```lua
S0 = 11
G[227] = S0
S0 = G[227]
__ret = f_000019E0(S0)
```

对应字节开头：

```text
0C 0B          push_i8 11
15 E3 00       pop_global 227
0F E3 00       push_global 227
02 E0 19 00 00 call 0x19E0
```

### 含义

- 把 `G[227]` 设为 `11`
- 调用 `f_000019E0(11)` 初始化名字栏样式

### 与前两个函数对比

| 函数 | `G[227]` 值 | 推测 |
|---|---:|---|
| `f_00000004` | 1 | 角色クロ对应的名字样式编号 |
| `f_00001918` | 99 | 主角/大雅类特殊名字样式编号 |
| `f_00000872` | 11 | 索尔/一磨/遠矢这一组对应的名字样式编号 |

所以这里再次证明：

> `G[227]` 更像“名字栏显示样式/角色显示类别编号”，而不是单纯的角色逻辑编号。

## 8. 与前三个名字显示函数体系的关系

将三个函数放在一起看，可以得到一个更完整的名字栏系统轮廓：

| 函数 | 样式/编号 | 名字分支 | 特征 |
|---|---:|---|---|
| `f_00000004` | `1` | `？？？ / クロ` | 普通角色公开二态 |
| `f_00001918` | `99` + 逻辑编号 `0` | `？？？ / ぼく / 大雅` | 主角/自称/真实名三态，样式编号与逻辑编号分离 |
| `f_00000872` | `11` | `？？？ / 一磨 / 遠矢 / ソル` | 多重身份别名四态 |

### 这说明什么？

说明 FVP 脚本中的“说话人名显示函数”很可能是按角色/角色组定制生成的，每个函数都编码了该人物在剧情里的：

- 名字显示样式
- 身份公开路径
- 别名集合
- 状态推进规则
- 上下文缓存规则

换句话说：

> 名字栏系统本身就是一个完整的“剧情身份管理子系统”。

而不是简单的“角色名字符串打印”。

## 9. 结论

这段 `f_00000872` 的具体作用可以概括为：

> **为“ソル”角色组设置名字栏样式，并根据参数在 `？？？ / 一磨 / 遠矢 / ソル` 四种说话人名之间切换；同时维护该角色组的名字公开状态计数器 `G[2011]`，刷新左侧名字栏，更新公共上下文变量，并缓存角色 11 对应的属性值。**

若要给它一个较贴切的高层名字，可以叫：

```text
ShowSpeakerName_SolAliasGroup
```

或者更明确些：

```text
ShowSpeakerName_ソル_一磨_遠矢
```

这段函数最大的研究价值在于，它证明了：

1. **名字栏函数可以支持多于 2 个名字分支**；
2. **一个角色组的身份公开过程可以是多层别名状态机**；
3. **状态变量（这里是 `G[2011]`）与名字显示分支之间存在明确联动**；
4. **FVP 的说话人名显示系统完全值得被抽象成一个独立的高层模块。**
