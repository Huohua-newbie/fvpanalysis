# SPEAK function block editor（实验版）

## 1. 目的

本目录提供一个面向 `SPEAK function` 的**第一版功能块系统编辑 GUI**：

- 固定函数模板中的大部分“固有结构”
- 把可编辑部分抽出来，让用户只改：
  - 名字栏样式编号（`G[227]`）
  - 状态变量编号（如 `G[2001] / G[2000] / G[2011]`）
  - 前置/后置状态推进集合与增量
  - 模式参数来自哪个 `push_stack`
  - 每种变体由哪个 `pushint` 值触发
  - 每个变体对应显示名、备用/问号名
  - `G[293] / G[294] / G[295]` 的上下文策略
  - 关键子函数地址

当前实验目标不是直接输出完整 `.hcb`，而是：

- 输出一个 `.lua` 文件：作为人类可读的 Lua-like 模板草稿
- 输出一个 `.tmp` 文件：作为该单个 function 的**原始 HCB 字节片段**

其中 `.tmp` 不是完整 HCB 文件，而是：

> **单个 function 的函数体字节序列（包含基于指定基址计算后的绝对 jmp/jz 地址）**

## 4. 当前模板抽象

当前 GUI 仅针对第一类名字显示函数，也就是这里所称的：

> **SPEAK function**

其抽象模板大致为：

```text
init_stack

G[227] = style_id
call setup_style(G[227])

[可选] 直接设置状态变量
[可选] 前置状态推进 if G[state] in {...} then += inc

根据 mode 参数：
  if trigger_1 -> SetAndPrintSpeakerName(name1, alt1) + SyncSpeakerState(...)
  if trigger_2 -> SetAndPrintSpeakerName(name2, alt2) + SyncSpeakerState(...)
  ...
  else       -> SetAndPrintSpeakerName(default_name, default_alt) + SyncSpeakerState(...)

[可选] 默认分支后置状态推进 if G[state] in {...} then += inc

G[29] = 1
G[293] = xxx / nil / conditional
G[294] = xxx
G[295] = GetCharacterProperty(property_id)
ret
ret
```

也就是说，它目前并不允许任意编辑所有指令，而是：

- **固定骨架**
- **开放关键参数**

这正符合你提出的“固定一些固有项目，将一些可编辑内容给出”的目标。

## 5. GUI 可编辑项

## 5.1 固定模板参数区

### 基础信息
- `函数名`
- `函数基址`
- `SPEAK块编号`
- `前序块大小(逗号分隔)`
- `参数数`

其中函数基址现在支持两种模式：

1. **手工指定**
2. **自动计算**

自动计算规则为：

- 当当前 `SPEAK` 块编号为 `1` 时，函数基址固定为 `4`
- 当当前块编号为 `N` 时：

```text
base_addr = 4 + sum(前面 N-1 个 SPEAK 块的大小)
```

这对应你提出的：

> 函数基址可以由已生成的 SPEAK 块数及每一块的大小计算；当当前 SPEAK 块编号为 1 时默认为 4。

### 样式与状态
- `G[227] 样式ID`
- `状态变量 G[]`
- `属性/角色ID`

### 栈帧来源
- `模式参数 push_stack`
- `G293参数 push_stack`
- `G294参数 push_stack`
- `G293保留开关 push_stack`

### 关键子函数地址
- `样式函数地址`（默认 `0x19E0`）
- `名字函数地址`（默认 `0x4CCDA`）
- `同步函数地址`（默认 `0x977B8`）
- `属性函数地址`（默认 `0x8D409`）

## 5.2 状态推进 / 同步策略区

### 前置状态推进
可配置：
- 是否启用
- 判定集合（例如 `0,2,4,6`）
- 增量（例如 `1`）

### 直接设置状态
可配置：
- 是否启用
- 固定赋值（例如 `7`）

### 默认分支后置状态推进
可配置：
- 是否启用
- 判定集合（例如 `0,1,4,5`）
- 增量（例如 `2`）

### 同步 ID 方式
支持：
- `style`：调用同步函数时使用 `G[227]`
- `fixed`：使用固定整数 ID

### G293 策略
支持：
- `always_arg`：总是把指定参数保存到 `G[293]`
- `conditional_keep`：当某个开关参数等于 1 时才保存，否则写 nil
- `nil`：总是写 nil

## 5.3 名字变体区（最重要的可编辑部分）

每个变体一行，可编辑：

- **触发 pushint**
  - 如 `-1` / `20` / `10` / `100`
  - 也支持 `default`
- **显示名**
  - 例如 `　　クロ　　`
- **备用/问号名**
  - 例如 `　 ？クロ 　`
  - 留空则视为 `nil`
- **备注**

这样就能直接表达：

- `-1 -> ？？？ / ?真实名`
- `20 -> ぼく`
- `10 -> 一磨`
- `100 -> 遠矢`
- `default -> ソル`

## 7. 输出文件格式

## 7.1 `.lua`
输出的是 **Lua-like 模板草稿**，其目的：

- 让人类快速检查这个 SPEAK function 的结构
- 便于和 `Sakura.lua` 风格对照
- 记录当前 GUI 配置对应的高层逻辑

注意：
- 它不是完整可回编译 HCB 的 IR
- 而是“函数块草稿 / 预览文件”

## 7.2 `.tmp`
输出的是该函数的 **原始 HCB 字节片段**，特点：

- 包含完整函数体字节序列
- 包含 `init_stack`
- 包含函数内部 `jz/jmp/call/push_string/...`
- 其中：
  - `call` 目标地址使用配置中的固定地址
  - `jmp/jz` 目标地址根据 `函数基址` 和本次汇编布局计算为绝对地址
  - 若启用了“自动计算函数基址”，则以块编号和前序块大小计算后的 `base_addr` 为准

注意：
- 它不是完整 `.hcb`
- 不能直接作为完整脚本运行
- 但可以作为后续拼接/回填进完整 HCB 的候选片段

## 8. 使用方式

在工作区根目录执行：

```bat
python fvp_analysis\result\hcbrebuild_test\speak_function_gui.py
```

基本流程：

1. 选择一个内置预设
2. 修改固定参数
3. 修改名字变体列表
4. 点击 **生成预览**
5. 在右侧检查 Lua-like 输出
6. 如需连续拼多个 SPEAK 块，可填写：
   - 当前块编号
   - 前序块大小列表
   然后点击 **刷新基址**
7. 点击 **导出 .lua + .tmp**
8. 选择输出文件名
9. 得到：
   - `xxx.lua`
   - `xxx.tmp`

如需保存当前配置，可点击：

- **保存配置 JSON**

## 9. 当前编码选择

你提出希望把写入编码明确收敛成：

- `sjis`
- `gbk`
- `utf8`

当前 GUI 已按这三档做统一按钮/下拉项约定，并在内部映射到 Python 编码器：

| GUI 选项 | 实际编码 |
|---|---|
| `sjis` | `shift_jis` |
| `gbk` | `gb18030` |
| `utf8` | `utf-8` |

这样做的原因是：

- `sjis` 对应原始 FVP 常见脚本编码
- `gbk` 对应传统中文移植/汉化工作流
- `utf8` 结合你提到的 `rfvp` 项目已有 GitHub 反馈，可作为实验性可用编译路径

因此当前导出时：

- `.tmp` 中的 `push_string`、名字字符串会按所选编码写入
- `.lua` 预览头也会记录当前 `write_encoding`

## 10. 当前限制

## 9.1 只支持第一类 SPEAK function

也就是当前只支持我们已经分析出的这类“说话人名显示函数”骨架。

尚不支持：
- 任意 CFG 编辑
- 任意 opcode 组合编辑
- 图像/音频/线程等其他功能块编辑

## 9.2 `.tmp` 不是完整 HCB

当前输出的 `.tmp` 只是：

- 一个函数体字节序列

它还没有自动：
- 插入到现有 HCB
- 修正其他函数地址引用
- 更新 sysdesc 或全局入口

因此当前更适合做：

> **功能块级原型设计 / 片段生成**

而不是完整回封。

## 9.3 Lua 预览是草稿，不是完整 IR

当前 `.lua` 只是“方便人读”的模板化输出，不是 `hcb_to_ir.py` 那种完整可逆 JSON IR。
